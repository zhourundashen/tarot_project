## 四、数据库表结构

### users 用户表
```sql
- id: 用户ID
- phone: 手机号
- password: 密码(bcrypt加密)
- nickname: 昵称
- credits: 积分
- vip_level: 会员等级(0普通/1月卡/2年卡)
- vip_expire_at: 会员到期时间
- created_at: 创建时间
```

### reading_records 占卜记录表
```sql
- id: 记录ID
- user_id: 用户ID
- spread_id: 牌阵ID
- question: 用户问题
- cards: 抽牌结果(JSON)
- interpretation: AI解读
- summary: 总结
- created_at: 创建时间
```

### orders 订单表
```sql
- id: 订单ID
- user_id: 用户ID
- amount: 金额
- type: 订单类型
- status: 状态
- created_at: 创建时间
```

### credit_logs 积分日志表
```sql
- id: 日志ID
- user_id: 用户ID
- amount: 变动数量
- type: 类型
- created_at: 创建时间
```

## 五、核心业务流程

```
欢迎页面
    │
    ▼
问题输入（可跳过）
    │
    ▼
选择牌阵（22种）
    │
    ▼
洗牌（含正逆位打乱）
    │
    ▼
切牌（三叠式）
    │
    ▼
抽牌（78张选N张）
    │
    ▼
翻牌 + AI首次解读
    │
    ▼
┌─────────────────────────┐
│      用户反馈            │
├─────────────────────────┤
│ 满意 → 生成总结 → 结束   │
│ 有问题 → 追问 → AI解读   │
│ 不满意 → 追问 → AI解读   │
│              ↑__________│
│                 (循环)   │
└─────────────────────────┘
```

## 六、配置说明

### js/config.js
```javascript
const CONFIG = {
    useBackend: true,        // 是否使用后端
    backendUrl: 'http://localhost:3000',
    // 前端直接调用时的配置(不推荐)
    apiKey: '...',
    apiUrl: 'https://open.bigmodel.cn/api/paas/v4/chat/completions',
    model: 'glm-4',
    provider: 'zhipu'
};
```

### server/.env
```
PORT=3000
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=你的密码
DB_NAME=tarot_db
JWT_SECRET=你的密钥
ZHIPU_API_KEY=你的智谱API Key
```
